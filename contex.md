Ошибка (no such package) возникла по двум причинам: во-первых, вы не обновили индексы пакетов перед установкой, а во-вторых, android-tools-adb — это название пакета в экосистеме Debian/Ubuntu. В OpenWrt (и ImmortalWrt) этот пакет называется просто adb.
Вот пошаговый план, как завести ADB на вашем роутере.
Шаг 1: Обновление индексов и установка пакета
Поскольку ImmortalWrt перешел на пакетный менеджер apk, сначала нужно актуализировать список доступных пакетов, а затем установить нужный:
apk update
apk add adb

(Опционально: если вы захотите в будущем настроить еще и раздачу интернета с этого телефона на роутер, вам также понадобятся драйверы: apk add kmod-usb-net-rndis kmod-usb-net-cdc-ncm).
Шаг 2: Запуск и авторизация устройства
После успешной установки пакета нужно запустить сервер ADB и проверить, видит ли он ваш телефон.
1. Убедитесь, что экран телефона разблокирован (это важно для первого подключения).
2. Выполните команду на роутере:
adb devices

При первом запуске этой команды демон ADB стартует автоматически. В этот момент на экране вашего Android-смартфона должно появиться всплывающее окно с просьбой разрешить отладку с этого компьютера (роутера) и показать RSA-отпечаток.
Обязательно поставьте галочку «Всегда разрешать с этого компьютера» и нажмите «ОК».
1. Запустите команду еще раз:
adb devices

Вывод должен выглядеть примерно так:
List of devices attached
XXXXXXXXX       device

(Если вместо device написано unauthorized, значит вы не подтвердили запрос на экране смартфона).
Шаг 3: Тестовый запрос SMS
Как только устройство перейдет в статус device, вы можете сразу же проверить чтение SMS прямо из терминала роутера:
adb shell "content query --uri content://sms/inbox --projection _id:address:body:date"

Эта команда должна выплюнуть в консоль содержимое ваших входящих сообщений. Если она отработала успешно, значит мост между BPI-R4 и смартфоном поднят корректно, и можно переходить к написанию скрипта парсинга.

Отлично, ADB поднялся, и вы успешно прочитали SMS (судя по тексту, от Сбера). Самое сложное позади.
Теперь нужно автоматизировать процесс: роутер должен по расписанию дергать телефон, парсить новые сообщения, отправлять их в Telegram и удалять с телефона.
Я адаптировал скрипт специально под особенности вывода команды content query в Android и урезанные утилиты среды ImmortalWrt/OpenWrt (BusyBox).
Шаг 1: Создание скрипта-парсера
Создайте файл скрипта на роутере:
vi /root/sms_telegram.sh

Вставьте туда следующий код. (Я убрал поле date из запроса, чтобы упростить парсинг текста, так как в самом тексте SMS могут быть запятые).
#!/bin/sh

TOKEN="ВАШ_ТОКЕН_БОТА"
CHAT_ID="ВАШ_CHAT_ID"

# Запрашиваем SMS без даты, чтобы не ломать парсинг, если в тексте есть запятые
adb shell "content query --uri content://sms/inbox --projection _id:address:body" > /tmp/sms_inbox.txt

# Читаем файл построчно
while IFS= read -r line; do
    # Пропускаем пустые строки
    [ -z "$line" ] && continue

    # Вытаскиваем данные с помощью sed
    SMS_ID=$(echo "$line" | sed -n 's/.*_id=\([0-9]*\).*/\1/p')
    SENDER=$(echo "$line" | sed -n 's/.*address=\([^,]*\), body=.*/\1/p')
    TEXT=$(echo "$line" | sed -n 's/.*body=\(.*\)/\1/p')

    # Если ID найден, значит строка валидная
    if [ -n "$SMS_ID" ]; then
        MESSAGE="✉️ <b>Новое SMS!</b>
<b>От:</b> $SENDER

$TEXT"

        # Отправляем в Telegram
        curl -s -X POST "https://api.telegram.org/bot$TOKEN/sendMessage" \
            -d chat_id="$CHAT_ID" \
            -d parse_mode="HTML" \
            -d text="$MESSAGE" > /dev/null

        # Удаляем отправленное SMS, чтобы не читать его по кругу
        adb shell "content delete --uri content://sms/_id --where \"_id=$SMS_ID\"" > /dev/null
    fi
done < /tmp/sms_inbox.txt

Сделайте файл исполняемым:
chmod +x /root/sms_telegram.sh

Вы можете протестировать скрипт прямо сейчас, запустив его руками: /root/sms_telegram.sh. Сообщения от Сбера должны прилететь в Telegram и исчезнуть из телефона.
Шаг 2: Настройка автоматизации через LuCI (Web-интерфейс)
В OpenWrt для запуска скриптов по расписанию используется стандартный Linux-планировщик cron, но в LuCI для него есть удобная графическая обертка.
1. Откройте браузер и зайдите в веб-интерфейс вашего роутера (обычно http://192.168.1.1).
2. В верхнем меню перейдите в System -> Scheduled Tasks (Система -> Планировщик заданий).
3. Откроется текстовое поле. Вставьте туда следующую строку: * * * * * /root/sms_telegram.sh  (Пять звездочек означают запуск каждую минуту).
4. Нажмите кнопку Save (Сохранить) в правом нижнем углу.
Важный нюанс OpenWrt: В некоторых сборках планировщик cron по умолчанию остановлен. Чтобы убедиться, что он работает:
1. В LuCI перейдите в System -> Startup (Система -> Загрузка).
2. Найдите в списке службу cron.
3. Убедитесь, что она включена (кнопка Enabled) и нажмите Start (Запустить), если она не запущена.
Шаг 3 (Важный): Настройка Android-смартфона
Android имеет агрессивную политику энергосбережения (Doze Mode). Когда экран смартфона гаснет, через некоторое время система может "усыпить" USB-порт, и команда adb devices на роутере начнет выдавать offline или перестанет видеть телефон.
Чтобы этого избежать:
1. Зайдите в Настройки разработчика на Android.
2. Включите опцию Оставить включенным (Stay awake) — Экран не будет переходить в спящий режим во время зарядки.
3. Убавьте яркость экрана смартфона на минимум и оставьте его лежать рядом с роутером. Роутер будет держать его заряженным, а экран никогда не заблокирует работу ADB.
