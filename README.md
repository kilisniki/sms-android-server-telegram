# SMS Android Server Telegram

Серверное приложение для чтения SMS и истории звонков (пропущенные, входящие) с подключенного по USB Android-смартфона и пересылки уведомлений в Telegram.

## Требования
- Любой Linux-хост (например, сервер, Raspberry Pi, роутер с OpenWrt) с Docker и Docker Compose
- Android-телефон с включенной отладкой по USB (USB Debugging)
- Телефон должен быть подключен по USB к хосту

## ВАЖНО: Конфликт ADB серверов
Встроенный ADB сервер на хост-машине (вашем сервере или роутере) **конфликтует** с ADB сервером внутри Docker контейнера. USB-порт устройства не может управляться одновременно двумя программами. 

Перед запуском контейнера необходимо остановить и удалить ADB на самом хосте, чтобы освободить USB интерфейс для Docker контейнера:

```bash
# Убедитесь, что процесс adb убит
killall adb

# Если ADB установлен как пакет (например, на Ubuntu/Debian)
sudo apt remove adb
# Или на OpenWrt/Alpine
apk del adb
```

## Настройка и Запуск

1. Склонируйте репозиторий:
```bash
git clone https://github.com/your/sms-android-server-telegram.git
cd sms-android-server-telegram
```

2. Скопируйте пример файла конфигурации:
```bash
cp .env.example .env
```

3. Отредактируйте `.env`:
Укажите токен вашего бота (от @BotFather) и ваш `CHAT_ID` (можно узнать у @userinfobot).

4. Запустите контейнер:
```bash
docker compose pull
docker compose up -d
```

5. **Авторизация телефона (Только в первый раз):**
   Когда контейнер запущен, на телефоне появится запрос авторизации отладки (RSA-отпечаток). Поставьте галочку "Всегда разрешать" и нажмите "ОК".
   
   Если запрос не появляется, вы можете "пнуть" ADB вручную из контейнера:
   ```bash
   docker exec -it sms-server adb devices
   ```
   В этот момент телефон 100% должен вывести запрос на экран.

## Особенности
- Используется легковесная база данных SQLite для предотвращения дубликатов сообщений (папка `data/`). Сами SMS на телефоне остаются нетронутыми.
- Автоматическое восстановление работы при переподключении кабеля.
- Уведомления о статусе подключения телефона.

## Энергосбережение (Важно для Android)
Чтобы Android не отключал USB-порт для экономии энергии при выключенном экране:
1. Зайдите в **Настройки разработчика** на телефоне.
2. Включите опцию **Оставить включенным (Stay awake)** (Экран не будет переходить в спящий режим во время зарядки).
3. Убавьте яркость экрана на минимум. Ваше Linux-устройство (роутер/сервер) будет постоянно держать телефон заряженным, а экран никогда не заблокирует работу ADB.
